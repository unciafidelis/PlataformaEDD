Las estructuras de datos primitivas son los tipos de datos básicos que los lenguajes de programación ofrecen por defecto:

  • Enteros (int)
  • Caracteres (char)
  • Flotantes (float, double)
  • Booleanos (true/false)

Las estructuras de datos no primitivas estas se basan en los tipos primitivos y ofrecen una organización más compleja.

  • Lineales
  • No lineales
  • Tablas

En las estructuras de datos lineales los elementos se organizan de forma secuencial y estos pueden ser:

  • Arreglos (Arrays): Colecciones de elementos del mismo tipo almacenados en posiciones contiguas.
  • Listas enlazadas: Una colección de nodos donde cada nodo apunta al siguiente.
  • Pilas (Stacks): Siguen el principio LIFO (Último en entrar, primero en salir).
  • Colas (Queues): Siguen el principio FIFO (Primero en entrar, primero en salir).

En las estructuras de datos no lineales los elementos se organizan en estructuras jerárquicas o relacionales. Estas pueden ser:

  • Árboles: Estructuras jerárquicas con un nodo raíz y nodos hijos.
  • Grafos: Conjuntos de nodos conectados por aristas, que pueden ser dirigidos o no dirigidos.
  • Tablas: Organizan los datos de forma estructurada para búsquedas rápidas. Las tablas hash utilizan una función hash (Una función hash es un algoritmo matemático que toma una entrada o tambien llamada llave/clave y la convierte en un valor de tamaño fijo, llamado código hash o digest) para acceder a los datos de manera eficiente.
